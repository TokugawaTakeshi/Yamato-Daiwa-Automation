Настройки обработки разметки

Обработка разметки может быть настроена в projectBuilding.markupProcessing файла конфигурации (yda.config.yaml по умолчанию): 

projectBuilding:

  markupProcessing:

    # Задайте настройки обработки разметки здесь в соответствии с этим руководством

Структура

projectBuilding.markupProcessing должны подчиняться следующим определениям TypeScript-типов: 

type MarkupProcessingSettings__FromFile__RawValid = Readonly<{
  common?: MarkupProcessingSettings__FromFile__RawValid.Common;
  linting?: MarkupProcessingSettings__FromFile__RawValid.Linting;
  staticPreview?: MarkupProcessingSettings__FromFile__RawValid.StaticPreview;
  entryPointsGroups: Readonly<{ [groupID: string]: MarkupProcessingSettings__FromFile__RawValid.EntryPointsGroup; }>;
  logging?: MarkupProcessingSettings__FromFile__RawValid.Logging;
}>;

namespace MarkupProcessingSettings__FromFile__RawValid {

  export type Common = Readonly<{
    periodBetweenFileUpdatingAndRebuildingStarting__seconds?: number;
    buildingModeDependent?: Readonly<{ [projectBuildingMode: string]: Common.BuildingModeDependent | undefined; }>;
  }>;

  export namespace Common {
    export type BuildingModeDependent = Readonly<{
      mustResolveResourceReferencesToRelativePaths?: boolean;
    }>;
  }


  export type Linting = Readonly<{
    presetFileRelativePath?: string;
    enable?: boolean;
  }>;


  export type StaticPreview = Readonly<{
    stateDependentPagesVariationsSpecificationFileRelativePath?: string;
    importsFromStaticDataFiles?: ReadonlyArray<StaticPreview.ImportFromStaticDataFile>;
    importsFromCompiledTypeScript?: Readonly<StaticPreview.ImportFromCompiledTypeScript>;
  }>;

  export namespace StaticPreview {

    export type ImportFromStaticDataFile = Readonly<{
      importedVariableName: string;
      fileRelativePath: string;
    }>;


    export type ImportFromCompiledTypeScript = Readonly<{
      typeScriptConfigurationFileRelativePath?: string;
      files: ReadonlyArray<ImportFromCompiledTypeScript.FileMetadata>;
    }>;

    export namespace ImportFromCompiledTypeScript {

      export type FileMetadata = Readonly<{
        importedNamespace: string;
        sourceFileRelativePath: string;
        outputDirectoryRelativePath: string;
        customOutputFileNameWithoutLastExtension?: string;
      }>;

    }

  }

  export type EntryPointsGroup =
      SourceCodeProcessingSettingsGenericProperties__FromFile__RawValid.EntryPointsGroup &
      Readonly<{
        HTML_Validation?: EntryPointsGroup.HTML_Validation;
        accessibilityInspection?: EntryPointsGroup.AccessibilityInspection;
        convertToHandlebarsOnNonStaticPreviewModes?: boolean;
        buildingModeDependent: Readonly<{ [projectBuildingMode: string]: EntryPointsGroup.BuildingModeDependent; }>;
      }>;

  export namespace EntryPointsGroup {

    export type BuildingModeDependent = SourceCodeProcessingSettingsGenericProperties__FromFile__RawValid.
        EntryPointsGroup.BuildingModeDependent &
        Readonly<{ outputCodeFormatting?: BuildingModeDependent.OutputCodeFormatting; }>;

    export namespace BuildingModeDependent {
      export type OutputCodeFormatting = Readonly<{ enable?: boolean; }>;
    }

    export type HTML_Validation = Readonly<{ disable?: boolean; }>;

    export type AccessibilityInspection = Readonly<{
      standard?: MarkupProcessingRestrictions.SupportedAccessibilityStandards;
      disable?: boolean;
    }>;

  }


  export type Logging = Readonly<{

    filesPaths?: boolean;
    filesCount?: boolean;
    partialFilesAndParentEntryPointsCorrespondence?: boolean;

    HTML_Validation?: Readonly<{
      starting?: boolean;
      completionWithoutIssues?: boolean;
    }>;

    accessibilityChecking?: Readonly<{
      starting?: boolean;
      completionWithoutIssues?: boolean;
    }>;

  }>;

}

Подробности

common — общие настройки

Тип
object
Обязательность
Необязательное

Определяет настройки, которые не связаны с конкретной группой точек входа.

periodBetweenFileUpdatingAndRebuildingStarting__seconds — Интервал между сохранением последнего файла разметки и началом пересборки

Тип
number
Значение по умолчанию
1

Определяет, сколько секунд должно пройти с момента сохранения последнего исходного файла разметки (то есть Pug-файла) до начала нового цикла обработки файлов разметки включая конвертацию из Pug в HTML. Актуально для инкрементальных режимов сборки (статическое превью, локальная разработка).

Даже если интегрированная среда разработки может сохранять несколько файлов за раз, на самом деле они сохраняются не одновременно, а с разницей в несколько миллисекунд. Кроме того, обнаружение изменения статуса файлов соответствующей утилитой также происходит с небольшой, но задержкой. Ввиду этого, без специальных мер будет одна пересборка на каждый сохранённый файл, что становится всё большим стрессом для аппаратного обеспечения по мере возрастания количества файлов. Чтобы это оптимизировать, необходимо подождать некоторое время после того, как один файл будет обновлён — возможно вслед за ним через несколько миллисекунд будет замечено сохранение последующих файлов.

В YDA на инкрементальных режимах сборки после сохранения одного Pug-файла запускается таймер ожидания пересборки, который истекает через periodBetweenFileUpdatingAndRebuildingStarting__seconds секунд. Если до истечения этого таймера не будет сохранено ни одного последующего Pug-файла, то запустится новый цикл обработки Pug-файлов. Если же до истечения этого таймера будет сохранён ещё один Pug-файл, то таймер ожидания будет перезапущен, и так до тех пор, пока в течение periodBetweenFileUpdatingAndRebuildingStarting__seconds не будет сохранено ни одного последующего Pug-файла.

Значения по умолчанию достаточно для начальной стадии разработки, когда обычно количество файлов невелико. Однако как только количество файлов станет порядка несколько десятков, может стать желательным увеличить значение periodBetweenFileUpdatingAndRebuildingStarting__seconds (возможны дробные числа, например 1.5). Поскольку по умолчанию обнаружение изменений в Pug-файлах и начало обработки каждого Pug-файла логируются, то Вы сможете заметить потребность в изменении значения periodBetweenFileUpdatingAndRebuildingStarting__seconds, если таковая возникнет.

buildingModeDependent — настройки, зависящие от режима сборки

Тип
object типа «ассоциативный массив»
Обязательность
Необязателен
Ключи
Должны быть режимом сборки проекта
Допустимые ключи
  • STATIC_PREVIEW
  • LOCAL_DEVELOPMENT
  • TESTING
  • STAGING
  • PRODUCTION
Тип значения
Объект
Схема значения
export type EntryPointsGroup =
    SourceCodeProcessingSettingsGenericProperties__FromFile__RawValid.EntryPointsGroup &
    Readonly<{
      HTML_Validation?: EntryPointsGroup.HTML_Validation;
      accessibilityInspection?: EntryPointsGroup.AccessibilityInspection;
      convertToHandlebarsOnNonStaticPreviewModes?: boolean;
      buildingModeDependent: Readonly<{ [projectBuildingMode: string]: EntryPointsGroup.BuildingModeDependent; }>;
    }>;

Настройки, зависящие от режима сборки проекта, однако общие для всех групп точек входа. Не следует путать с настройками entryPointsGroups.entryPointsGroups.[GROUP_ID.buildingModeDependent], которые зависят как от группы точек входа, так и от режима сборки проекта.

logging — настройки логирования

Тип
object
Обязательность
Необязательное

Определяет настройки логирования в терминал.

filesPaths — пути файлов

Тип
boolean
Значение по умолчанию
true

Выводит в консоль пути обрабатываемых файлов, относительные директории, в которой находится файл настроек YDA.

Поскольку при начале обработке Pug-файла должна быть какая-то обратная связь, то по умолчанию это логирование включено.

filesCount — число файлов

Тип
boolean
Значение по умолчанию
false

Выводит в количество файлов, которое будет обработано в одном цикле сборки (на режимах инкрементальной сборки таких циклов может быть сколь угодно много, и каждый раз с разным количеством файлов).

По умолчанию имеет значение false, так как если файлов мало, то обычно достаточно вывести лишь пути обрабатываемых файлов, а если файлов много, мало кто захочет проверять правильность этого числа.
